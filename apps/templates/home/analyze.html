How to read PostgreSQL column statistics
These values usually come from pg_stats after ANALYZE. The planner uses them to estimate how many rows a predicate will return.
Selected column statistic
boarding_time [<]: n_distinct=-0.9938, null_frac=0.0062, mcv_count=0, histogram_bounds=101
Summary for the selected column

This column is almost always populated, has very high cardinality, has no dominant repeated values, and has a histogram available for range selectivity estimates.

The table below remains as a pedagogical reference for each PostgreSQL statistic shown in the selected line.
Statistic Meaning How to interpret it Planner impact
n_distinct Estimated number of distinct non-null values. Positive value = absolute estimated count. Negative value = fraction of the table row count. For example, -0.9938 means approximately 0.9938 × reltuples distinct values. Used for equality selectivity, joins, grouping, and deciding whether an index is likely to be selective.
null_frac Estimated fraction of rows where the column is NULL. 0.0062 means about 0.62% of rows are null. Used for IS NULL, IS NOT NULL, and as a correction factor for other predicates.
mcv_count Number of Most Common Values stored for the column. 0 means PostgreSQL did not keep a list of dominant values, often because values are highly unique or no value is frequent enough. When MCVs exist, equality predicates on common values can be estimated more accurately than with a uniform assumption.
histogram_bounds Number of histogram boundary values stored. 101 usually represents 100 histogram buckets. It is common when default_statistics_target = 100. Used mainly for range predicates such as <, <=, >, >=, and BETWEEN.
[<] Ordering operator used to build or interpret the histogram. For timestamps, it means values are ordered chronologically from older to newer. Relevant for estimating inequalities and range filters.
Typical reading
High n_distinct plus mcv_count=0 means values are spread out. Equality predicates are expected to match very few rows; range predicates depend mostly on the histogram.
When to be careful
Statistics are estimates from a sample. If the table changed significantly, run ANALYZE. If estimates remain poor, consider increasing statistics with ALTER TABLE ... ALTER COLUMN ... SET STATISTICS.
